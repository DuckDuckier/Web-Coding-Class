CSS Hacks por Navegador

Navegador
Hack usado
¿Por qué funciona?
Chrome / Safari
-webkit-text-stroke
Usa motor WebKit, que interpreta propiedades con prefijo -webkit-.
Firefox
@-moz-document
Solo Firefox reconoce esta regla especial para aplicar estilos condicionales.
Internet Explorer
@media all and (-ms-high-contrast)
IE interpreta esta media query exclusiva para su modo de alto contraste.
Safari
@supports (-webkit-appearance:none)
Safari responde a esta combinación de media query y soporte condicional.

Reflexión final

Los hacks son útiles para resolver problemas de compatibilidad entre navegadores, especialmente en proyectos donde se necesita una visualización consistente. Sin embargo, no deben usarse como solución permanente, ya que pueden volverse obsoletos o generar más problemas a largo plazo.

Lo ideal es escribir CSS compatible con todos los navegadores modernos, usar herramientas como normalize.css o autoprefixer, y aplicar hacks solo cuando sea estrictamente necesario.
