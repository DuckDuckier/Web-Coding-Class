¿Por qué los <iframe> pueden ser inseguros?

1. Política de mismo origen

Los navegadores impiden que accedas al contenido de un iframe si proviene de otro dominio. Esto protege la privacidad del usuario.

2. Clickjacking

Un atacante puede ocultar un iframe y superponer botones falsos. El usuario cree que hace clic en algo legítimo, pero está interactuando con el contenido del iframe.

3. Contenido malicioso

Si permites que se cargue cualquier URL en un iframe, podrías mostrar páginas con virus, phishing o scripts peligrosos.

Ejemplo: A continuación se intenta cargar Google en un iframe. Muchos sitios bloquean esto por seguridad.

4. Encabezados de seguridad

Muchos sitios usan encabezados como X-Frame-Options o Content-Security-Policy para evitar ser incrustados en otros sitios.

¿Cómo protegerse?

  • Usa el atributo sandbox para limitar lo que puede hacer el iframe.
  • No cargues contenido externo que no controles.
  • Evita usar iframes para formularios o datos sensibles.
