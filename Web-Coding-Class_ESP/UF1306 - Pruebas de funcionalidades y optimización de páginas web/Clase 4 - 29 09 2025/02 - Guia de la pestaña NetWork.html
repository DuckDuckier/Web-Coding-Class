Guía de la pestaña Network

Explorando las herramientas F12 del navegador

¿Qué es la pestaña Network?

La pestaña Network permite monitorizar en tiempo real todas las solicitudes que realiza una página web: archivos HTML, CSS, JS, imágenes, fuentes, vídeos, peticiones AJAX, etc. Es como ver el tráfico que fluye entre el navegador y el servidor.

¿Qué puedes ver y hacer?

  • Registro de solicitudes: Lista de todos los recursos solicitados al cargar la página.
  • Filtrado por tipo: Imágenes, scripts, documentos, XHR, fuentes, etc.
  • Detalles de cada solicitud: Encabezados, contenido, tiempos, cookies.
  • Errores de carga: Detecta recursos que no se han cargado correctamente.
  • Control de caché: Simula una primera visita desactivando la caché.
  • Simulación de red lenta: Emula conexiones 3G, offline, etc.

Detalles de cada solicitud

Sección Qué muestra
Headers Encabezados HTTP de solicitud y respuesta (MIME, CORS, caché)
Preview Vista previa del contenido (HTML, JSON, imagen, etc.)
Response Contenido completo recibido
Timing Desglose de tiempos: DNS, conexión, espera, descarga
Cookies Cookies asociadas a la solicitud

Errores comunes

  • 404 Not Found: El archivo no existe.
  • 403 Forbidden: Acceso denegado por el servidor.
  • net::ERR_NAME_NOT_RESOLVED: El dominio no se puede resolver.
  • CSP blocked: Bloqueado por política de seguridad (Content Security Policy).

¿Cómo usarlo en tus ejercicios?

  • Verifica si los componentes se cargan correctamente.
  • Detecta errores de integración (imágenes rotas, PDF 404, scripts no definidos).
  • Evalúa el rendimiento y la accesibilidad de cada recurso.
  • Documenta los resultados como parte del informe técnico del alumno.

Para más información, consulta la documentación oficial de Microsoft Edge o Chrome DevTools.
