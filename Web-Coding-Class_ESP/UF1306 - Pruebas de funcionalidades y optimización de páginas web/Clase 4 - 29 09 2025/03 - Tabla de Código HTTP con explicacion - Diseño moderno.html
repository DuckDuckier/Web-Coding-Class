Códigos de Estado HTTP

Guía técnica con ejemplos prácticos para desarrolladores

1xx – Informativos

Código Descripción Ejemplo práctico
100 Continue – El cliente puede continuar con la solicitud. Antes de enviar un archivo grande, el navegador espera confirmación del servidor para continuar.
101 Switching Protocols – El servidor acepta cambiar el protocolo. Una app web solicita cambiar de HTTP a WebSocket para mantener una conexión en tiempo real.
102 Processing – El servidor está procesando la solicitud (WebDAV). El servidor indica que está trabajando en una operación compleja como copiar múltiples archivos.
103 Early Hints – El servidor sugiere recursos a precargar. El navegador comienza a cargar `style.css` y `main.js` antes de recibir el HTML completo.

2xx – Peticiones correctas

Código Descripción Ejemplo práctico
200 OK – Solicitud exitosa. La página se carga correctamente y todos los recursos (CSS, JS, imágenes) se muestran sin errores.
201 Created – Recurso creado. Al enviar un formulario de registro, el servidor crea el nuevo usuario y devuelve este código.
204 No Content – Sin contenido. Una petición AJAX se procesa correctamente pero no necesita devolver datos.
206 Partial Content – Parte del recurso entregado. Al descargar un archivo grande, el navegador solicita solo una parte (por ejemplo, del 50% al 75%).

3xx – Redirecciones

Código Descripción Ejemplo práctico
301 Moved Permanently – Redirección permanente. Una URL antigua redirige automáticamente a una nueva, como cuando se cambia la estructura de un blog.
302 Found – Redirección temporal. Una página de login redirige temporalmente al panel de usuario tras iniciar sesión.
304 Not Modified – El recurso no ha cambiado. El navegador usa la versión en caché porque el archivo no ha sido modificado desde la última visita.

4xx – Errores del cliente

Código Descripción Ejemplo práctico
400 Bad Request – Solicitud mal formada. Un formulario envía datos incompletos o con formato incorrecto (por ejemplo, email sin “@”).
401 Unauthorized – No autorizado. Intentas acceder a una zona privada sin haber iniciado sesión.
403 Forbidden – Acceso denegado. El servidor bloquea el acceso a un archivo protegido, aunque estés autenticado.
404 Not Found – Recurso no encontrado. La URL apunta a una página que no existe o fue eliminada.
429 Too Many Requests – Demasiadas solicitudes. Has recargado la página o enviado formularios demasiadas veces en poco tiempo.

5xx – Errores del servidor

Código Descripción Ejemplo práctico
500 Internal Server Error – Error interno. El servidor tiene un fallo en el código PHP o en la base de datos.
502 Bad Gateway – Puerta de enlace incorrecta. Un proxy o CDN no puede comunicarse con el servidor de origen.
503 Service Unavailable – Servicio no disponible. El servidor está sobrecargado o en mantenimiento.
504 Gateway Timeout – Tiempo de espera agotado. El servidor no responde a tiempo a través de una puerta de enlace o proxy.
