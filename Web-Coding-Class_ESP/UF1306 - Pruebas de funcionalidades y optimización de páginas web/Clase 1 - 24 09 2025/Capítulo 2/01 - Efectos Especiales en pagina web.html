UF1306 - Capítulo 2: Efectos especiales en páginas web

Pruebas de funcionalidades y optimización de páginas web

1. Introducción

Los efectos especiales en páginas web permiten mejorar la experiencia del usuario, aportando dinamismo, estética y usabilidad. Estos efectos pueden aplicarse a imágenes, texto, marcos, ventanas o mediante tecnologías como CSS, JavaScript y librerías externas (ej. jQuery).

2. Trabajar con imágenes

Las imágenes son un recurso fundamental en el diseño web. Podemos aplicar efectos de sustitución, mostrar varias imágenes o secuencias animadas.

2.1 Formatos de imágenes

  • JPEG: compresión con pérdida, ideal para fotografías.
  • PNG: compresión sin pérdida, soporta transparencia.
  • GIF: soporta animaciones simples, 256 colores.
  • SVG: formato vectorial, escalable sin pérdida de calidad.

2.2 Selección de imágenes

  • Vectoriales: basadas en fórmulas matemáticas, escalables (ej. logotipos en SVG).
  • Bitmap: basadas en píxeles, pierden calidad al escalar (ej. JPG, PNG).
  • Resolución: número de píxeles por pulgada (ppi). A mayor resolución, mayor calidad.
  • Profundidad de color: número de bits usados para representar el color (8, 16, 24 bits).
  • Modos de color:
    • RGB (aditivo): mezcla de rojo, verde y azul. Usado en pantallas.
    • CMYK (sustractivo): mezcla de cian, magenta, amarillo y negro. Usado en impresión.

2.3 Optimización de imágenes

Optimizar imágenes mejora la velocidad de carga de la web:

  • Escalar: reducir o ampliar sin deformar.
  • Optimizar: elegir el formato adecuado (PNG para transparencias, JPG para fotos, GIF para animaciones simples).

2.4 Implementación de código con varias imágenes

Ejemplo de sustitución de imagen al pasar el ratón:

<img src="imagen1.jpg" 
     onmouseover="this.src='imagen2.jpg'" 
     onmouseout="this.src='imagen1.jpg'">

Ejemplo con jQuery para mostrar secuencia de imágenes:

<div id="galeria">
  <img src="img1.jpg" alt="Imagen 1">
  <img src="img2.jpg" alt="Imagen 2" style="display:none">
</div>

<script>
let index = 0;
setInterval(function(){
  const imgs = $("#galeria img");
  imgs.hide();
  index = (index + 1) % imgs.length;
  imgs.eq(index).fadeIn();
}, 2000);
</script>

3. Trabajar con texto

3.1 Creación de textos mejorados y con movimiento

Podemos aplicar sombras, degradados y animaciones CSS:

h1 {
  text-shadow: 2px 2px 5px gray;
  background: linear-gradient(to right, red, orange, yellow);
  -webkit-background-clip: text;
  color: transparent;
  animation: mover 3s infinite alternate;
}

@keyframes mover {
  from { transform: translateX(0); }
  to { transform: translateX(50px); }
}

3.2 Implementación de efectos

Ejemplo con jQuery:

$("#titulo").fadeIn(2000).slideUp(2000).slideDown(2000);

3.3 Adecuación de efectos a la página web

Los efectos deben ser coherentes con el diseño y no distraer al usuario. Se recomienda usarlos con moderación.

4. Trabajar con marcos

4.1 Dónde utilizar los marcos

Los marcos (<iframe>) permiten incrustar contenido externo, como vídeos de YouTube o mapas.

4.2 Limitaciones de los marcos

Pueden afectar la accesibilidad, el SEO y la seguridad.

4.3 Alternativas a los marcos

Uso de APIs, incrustación con JavaScript o componentes modernos.

5. Trabajar con ventanas

5.1 Creación de varias ventanas

<script>
function abrirVentana() {
  window.open("https://ejemplo.com", "nueva", "width=400,height=300");
}
</script>
<button onclick="abrirVentana()">Abrir ventana</button>

Los navegadores pueden bloquear pop-ups, por lo que hay que usarlos con cuidado.

5.2 Interactividad entre varias ventanas

Se puede comunicar entre ventanas con window.opener o APIs modernas como postMessage.

6. Otros efectos

6.1 Efectos con HTML (Canvas)

<canvas id="miCanvas" width="200" height="100"></canvas>
<script>
const c = document.getElementById("miCanvas");
const ctx = c.getContext("2d");
ctx.fillStyle = "blue";
ctx.fillRect(20, 20, 150, 50);
</script>

6.2 Efectos con CSS

.caja {
  width: 100px; height: 100px;
  background: red;
  transition: transform 0.5s;
}
.caja:hover {
  transform: rotate(45deg) scale(1.2);
}

6.3 Efectos con capas (jQuery)

<div id="panel" style="display:none; padding:20px; background:#eef;">
  <p>Contenido del panel</p>
</div>
<button id="btnToggle">Alternar</button>

<script>
$("#btnToggle").click(() => $("#panel").fadeToggle(500));
</script>

7. Resumen y ejercicios

7.1 Resumen

  • Imágenes: formatos (JPEG, PNG, GIF, SVG), selección según uso, optimización y efectos de sustitución/secuencias.
  • Texto: mejoras visuales con CSS (sombras, degradados) y movimiento con keyframes o jQuery.
  • Marcos: uso de iframes, sus limitaciones y alternativas modernas (APIs, componentes).
  • Ventanas: apertura controlada, bloqueadores de pop-ups, comunicación entre ventanas.
  • Otros efectos: Canvas para gráficos, CSS para transformaciones/animaciones, jQuery para efectos en capas y encadenamientos.

7.2 Ejercicios

  1. Imágenes de sustitución: crea una tarjeta que cambie la imagen al pasar el ratón y vuelve a la original al salir.
  2. Secuencia jQuery: implementa una mini-galería que alterna entre 3 imágenes con fadeIn/fadeOut cada 2s.
  3. Título animado: diseña un título con degradado y animación CSS usando @keyframes (movimiento horizontal suave).
  4. Iframe responsable: incrusta un vídeo con <iframe>, añade estilos responsivos y explica una limitación y su alternativa.
  5. Pop-up controlado: crea un botón que intente abrir una ventana con window.open; si está bloqueada, muestra un aviso en la página.
  6. Panel con efectos: construye un panel oculto que se muestre con fadeIn y se oculte con slideUp; añade un callback que cambie el fondo.
  7. Canvas básico: dibuja un rectángulo y un círculo; añade un pequeño bucle de animación para cambiar su tamaño con requestAnimationFrame.
  8. Encadenamiento jQuery: anima tamaño → opacidad → posición y muestra “Animación completa” al terminar (callback).

7.3 Recomendaciones finales

  • Propósito: usa efectos para mejorar comprensión o feedback, no por decoración excesiva.
  • Rendimiento: optimiza imágenes, limita animaciones pesadas y prueba en móviles.
  • Accesibilidad: evita contenido parpadeante rápido, cuida contraste y ofrece alternativas si el efecto oculta información.
