Arrow Functions en JavaScript

1. ¿Qué es una Arrow Function?

Una Arrow Function es una forma más concisa de escribir funciones en JavaScript.  
Utiliza la sintaxis: (param1, param2) => { sentencias }
Las Arrow Functions eliminan la necesidad de usar function y tienen un comportamiento de this más predecible. Son ideales para funciones simples y callbacks.

2. Sumar dos números

const suma = (a, b) => a + b;
console.log(suma(3, 4)); // 7
Como la función tiene una sola expresión, se puede omitir {} y return. La flecha => separa los parámetros del cuerpo de la función.

3. Elevar al cuadrado

const cuadrado = num => num ** 2;
console.log(cuadrado(5)); // 25
Cuando hay un solo parámetro, se pueden omitir los paréntesis. La expresión num ** 2 devuelve el cuadrado del número.

4. Doblar un número

const doble = num => {
  const resultado = num * 2;
  return resultado;
};

console.log(doble(6)); // 12
Cuando el cuerpo de la función tiene varias líneas, se deben usar {} y return explícitamente. Esto permite realizar operaciones más complejas dentro de la función.

5. Ventajas y desventajas

Ventajas:
- Sintaxis más corta y legible.
- Contexto de this más predecible.
- Ideal para callbacks y funciones anónimas.

Desventajas:
- No se pueden usar como constructores.
- No tienen arguments propio.
- Pueden dificultar la lectura si se abusan.
Las Arrow Functions son poderosas, pero deben usarse con criterio. No reemplazan completamente a las funciones tradicionales.

6. Buenas prácticas con Arrow Functions

- Usa const para declarar funciones que no se reasignarán.
- Prefiere la sintaxis de una sola línea cuando sea posible.
- Usa nombres descriptivos para las funciones.
- Evita mutar variables externas dentro de la función.
- Verifica el contexto de this si estás dentro de objetos o clases.
Estas prácticas ayudan a mantener tu código limpio, legible y libre de errores sutiles. Las Arrow Functions brillan cuando se usan con intención y claridad.
