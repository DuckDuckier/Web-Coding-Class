Comparación: Ordinograma Clásico vs Flowgorithm

Este recurso ayuda a entender cómo se representan los algoritmos en papel (ordinograma clásico) y cómo se implementan en Flowgorithm.

¿Qué es un Ordinograma?

Es un diagrama de flujo que usa símbolos estándar para representar procesos, decisiones, entradas y salidas.

Ejemplo: Óvalo = Inicio/Fin, Rectángulo = Proceso, Rombos = Decisión, Paralelogramo = Entrada/Salida

¿Qué es Flowgorithm?

Es una herramienta educativa que permite crear y ejecutar algoritmos visualmente. Usa símbolos similares, pero exige detalles técnicos como tipos de datos, declaraciones y sintaxis precisa.

Diferencias clave: Flowgorithm requiere declarar variables, usar operadores específicos (como & para concatenar), y sigue reglas de ejecución.
