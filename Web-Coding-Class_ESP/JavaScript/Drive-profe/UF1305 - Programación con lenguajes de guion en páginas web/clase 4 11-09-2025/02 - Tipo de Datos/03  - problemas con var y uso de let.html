Problemas con var y uso de let

Aprende a escribir código más seguro y moderno en JavaScript

¿Por qué evitar var?

La palabra clave var fue la primera forma de declarar variables en JavaScript, pero tiene comportamientos que pueden causar errores. Aquí te explicamos los tres principales:

1. Ámbito poco claro

Las variables con var no se limitan al bloque donde se declaran. Esto significa que pueden ser accesibles fuera de estructuras como if o for, lo que puede generar confusión.

if (true) {
  var mensaje = "Hola";
}
console.log(mensaje); // "Hola" — aunque está fuera del bloque

2. Elevación (hoisting)

Las variables var se mueven automáticamente al inicio del código, pero sin su valor. Esto puede hacer que el código se comporte de forma inesperada.

console.log(nombre); // undefined
var nombre = "Lucía";

El navegador interpreta esto como:

var nombre;
console.log(nombre); // undefined
nombre = "Lucía";

3. Permite redeclaración

Con var, puedes declarar la misma variable varias veces sin que el navegador te avise. Esto puede sobrescribir valores sin darte cuenta.

var edad = 20;
var edad = 30; // No hay advertencia
console.log(edad); // 30

¿Cuándo usar let?

let se usa cuando necesitas una variable que pueda cambiar su valor, pero que esté limitada al bloque donde se declara. Es ideal para bucles, condiciones y funciones.

for (let i = 0; i < 3; i++) {
  console.log("Número:", i);
}
console.log(i); // Error: i no está definido fuera del bloque

También puedes usar let para controlar valores que cambian según la lógica del programa:

let resultado = 0;

if (true) {
  let factor = 5;
  resultado = factor * 2;
}

console.log(resultado); // 10
console.log(factor); // Error: factor no está definido fuera del bloque
Consejo: Usa let para variables que cambian y const para valores fijos. Evita var para escribir código más claro y seguro.
