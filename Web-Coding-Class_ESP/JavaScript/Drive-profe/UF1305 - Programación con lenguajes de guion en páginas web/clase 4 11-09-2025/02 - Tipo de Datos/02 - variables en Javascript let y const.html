¿Por qué usar let y const?

Evita errores y escribe código más seguro en JavaScript

En JavaScript, existen tres formas de declarar variables: var, let y const. Aunque var fue la primera en existir, hoy se recomienda usar let y const porque ofrecen mayor seguridad y claridad en el código.

Problemas con var

  • Ámbito poco claro: Las variables con var no respetan los bloques (if, for, etc.), lo que puede causar errores.
  • Elevación (hoisting): Se mueven al inicio del código automáticamente, pero sin su valor, lo que puede generar confusión.
  • Permite redeclaración: Puedes declarar la misma variable varias veces sin que el navegador lo advierta.

Ventajas de let y const

  • Ámbito de bloque: Solo existen dentro del bloque donde se crean, lo que evita errores.
  • Mayor control: El navegador avisa si intentas redeclarar una variable.
  • const protege valores: Ideal para datos que no deben cambiar, como configuraciones o constantes.

Ejemplo comparativo

// var permite esto sin error
var nombre = "Ana";
var nombre = "Luis"; // No hay advertencia

// let y const evitan esto
let edad = 25;
let edad = 30; // Error: ya está declarada

const PI = 3.1416;
PI = 3.14; // Error: no se puede cambiar

¿Cuándo usar cada uno?

  • Usa const si el valor no va a cambiar.
  • Usa let si el valor puede cambiar durante la ejecución.
  • Evita var salvo que estés trabajando con código antiguo.
Consejo: Usar let y const te ayuda a escribir código más claro, seguro y moderno. ¡Hazlo parte de tu rutina desde el inicio!
