Lámina visual: símbolos de un ordinograma

Guía sencilla para reconocer cada símbolo y su función.

Elipse — Inicio / Fin
Inicio / Fin
Flujo

Para qué sirve: marca el comienzo o el final del algoritmo.

Ejemplo: “Inicio” al empezar, “Fin” al terminar.

Paralelogramo — Entrada / Salida
Leer / Escribir
I/O

Para qué sirve: cuando el programa recibe datos o muestra información.

Ejemplos: “Leer edad”, “Mostrar resultado”.

Rectángulo — Proceso
Cálculo / Acción
Acción

Para qué sirve: realizar cálculos o asignaciones.

Ejemplo: “suma = a + b”.

Rombo — Decisión (Sí/No)
¿Condición?
Rama

Para qué sirve: hacer una pregunta y elegir un camino según la respuesta.

Ejemplo: “¿Edad ≥ 18?” → Sí / No.

Rectángulo con doble borde — Subproceso / Función
Llamar Subproceso
Módulo

Para qué sirve: llamar a un bloque reutilizable (un “mini-programa”).

Ejemplo: “CalcularPromedio()”.

Conector — Continuación
Salto

Para qué sirve: continuar el flujo cuando no hay espacio o se sigue en otra página.

Usa letras o etiquetas: “A”, “B”…

Flechas — Dirección del flujo

Para qué sirve: muestran el orden de las acciones. Van de un símbolo al siguiente.

Consejo: evita cruces innecesarios para que el diagrama se entienda mejor.
