Proyecto Web Integrador

Aplicación práctica de HTML, CSS y JavaScript

Objetivo del Proyecto

Aplicar de forma práctica todos los conocimientos adquiridos en HTML, CSS y JavaScript mediante la creación de una aplicación web funcional, estructurada y visualmente atractiva. Cada alumno podrá elegir libremente el tipo de proyecto que desea realizar (por ejemplo: página personal, tienda virtual, blog, portafolio, herramienta interactiva, etc.), siempre que cumpla con los requisitos técnicos establecidos.

El sitio web debe incluir al menos cuatro secciones o páginas simuladas:

  • Inicio
  • Quiénes somos
  • Contenido o servicios
  • Contacto (con formulario funcional)

Versión estándar (obligatoria)

1. HTML

  • Uso de estructura semántica: <header>, <nav>, <main>, <section>, <article>, <form>, <footer>
  • Navegación funcional entre páginas o secciones
  • Formulario con los siguientes campos:
    • Nombre completo
    • Correo electrónico
    • Contraseña
    • Confirmación de contraseña
    • Fecha de nacimiento
    • Género (radio buttons)
    • País (select)
    • Aceptar términos y condiciones (checkbox)
    • Botón de envío y botón de reinicio

2. CSS

  • Estilos personalizados: tipografía, colores, espaciado, bordes y sombras
  • Efectos visuales al enfocar campos (:focus)
  • Diseño responsive con media queries
  • Estilos diferenciados para mensajes de error y éxito
  • Estilo coherente entre todas las páginas del sitio

3. JavaScript

  • Validación del formulario:
    • Campos completos
    • Formato de correo electrónico
    • Contraseñas coincidentes
    • Casilla de términos aceptada
  • Mensajes dinámicos de error o éxito
  • Eventos: onfocus, onblur, onchange, onsubmit, onreset, onclick, oninput
  • Uso de localStorage para guardar datos
  • Mostrar resumen de datos ingresados
  • Botón para borrar datos almacenados

Versión avanzada (opcional)

  • Modo oscuro con JavaScript y localStorage
  • Validación de edad mínima (ej. mayor de 18 años)
  • Animaciones con CSS (transiciones, transformaciones)
  • Barra de progreso en el formulario
  • Simulación de envío con fetch()
  • Componentes reutilizables: tarjetas, menús, modales
  • Secciones dinámicas que se actualicen en tiempo real
  • Uso de JSON.stringify() y JSON.parse()

Evaluación

  • ✔️ Correcta estructura semántica en HTML
  • ✔️ Estilos CSS bien aplicados y diseño responsive
  • ✔️ Validaciones funcionales en JavaScript
  • ✔️ Uso adecuado de eventos y almacenamiento local
  • ✔️ Código limpio, comentado y organizado
  • ✔️ Creatividad y coherencia en el diseño y funcionalidad
  • ✔️ Navegación clara entre páginas o secciones
